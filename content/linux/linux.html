Linux - Basic

Introduction to Linux

This course introduces you to the world of Linux—an open-source operating system that powers everything from smartphones to servers. Linux is a powerful, open-source operating system widely used for servers, development, cybersecurity, and personal computing. Based on the Linux kernel, it provides stability, security, and flexibility, making it a preferred choice for tech professionals and enthusiasts.

Whether you're new to Linux or looking to enhance your expertise, exploring its various distributions and tools will give you deeper control over your computing environment.

Linux offers several security-focused distributions designed for penetration testing, incident response, and digital forensics. Among the most well-known are Kali Linux, which is widely used for ethical hacking and security assessments, and Parrot OS, offering a balance between privacy and offensive security tools. Exegol, a relatively newer contender, is an advanced container-based environment built for cybersecurity operations, making it easy to set up dedicated workspaces for various security tasks. Other notable distributions include BlackArch, tailored for penetration testers, and Tails, which emphasizes anonymity and secure communications.

For Windows users, WSL (Windows Subsystem for Linux) provides a convenient way to run Linux security tools directly inside Windows, making it easier to use distributions like Kali Linux without needing a separate virtual machine or dual-boot setup. With WSL, professionals can integrate powerful security features while maintaining compatibility with Windows-based workflows.

Linux File System

Imagine your computer is like a giant digital house. It has different rooms where things are stored, and every room has a special purpose. Linux organizes everything inside a big tree starting from the main root.

Root Directory / - The Starting Point

The root directory is like the front door to your digital house. Every other folder is inside it.
Example: cd / takes you to the main root.

/bin - Toolbox for Commands

Inside /bin, you find important tools to help manage your computer, like ls (list files) and bash (your command-line helper).
Try: ls /bin to see the toolbox.

/etc - The Rulebook

The /etc folder holds important settings for your computer, like passwords and system rules.
Try: cat /etc/passwd to see user details.

/home - Your Personal Space

The /home folder is like your bedroom. Each person using the computer has their own folder to store pictures, documents, and files.
Try: cd /home and ls to see different user rooms.

/var - Logbook and Records

The /var folder keeps logs, or the history of what happens in your computer. It's like a diary tracking important events.
Try: ls /var/log to check logs.

/usr - The App Storage

The /usr folder is like a warehouse where installed apps and useful programs are stored.
Try: ls /usr/bin | head to peek inside.

/tmp - The Temporary Space

The /tmp folder is like a junk drawer in your house. It holds files that only need to exist for a short time before they are deleted. If your computer needs a quick place to put something while working, it goes here.
Try: ls /tmp to see what's temporarily stored.

/opt - Special Storage for Extra Apps

The /opt folder is where special extra software is stored. If you install special programs that don't come with Linux by default, they might live here.
Try: ls /opt to see if your computer has optional software installed.

Try answering these questions with your terminal:

  • Where do system logs get saved?
  • What command lists everything in /etc?
  • How do you go inside your personal home folder?
  • What is stored in /tmp?
  • What kind of files can be found in /opt?

Essential Linux Commands

Linux is a foundational tool in cybersecurity and ethical hacking, offering powerful commands for system management, monitoring, and analysis. Below are commonly used commands and what they do.

System & User Management

  • whoami – Shows the current logged-in user
  • id – Displays user and group IDs
  • passwd – Changes the current user's password
  • sudo – Executes a command as another user (usually root)
  • groups – Lists the groups the user belongs to

File & Permission Management

  • ls – Lists files and directories
  • find – Searches for files and directories
  • chmod – Modifies file permissions
  • chown – Changes file ownership
  • stat – Displays detailed file information
  • cat – Displays file contents
  • tr – Translates or deletes characters from input
  • awk – Pattern scanning and processing language, useful for data extraction and reporting

Network & Monitoring

  • ifconfig / ip – Displays network configuration
  • netstat – Shows network connections and routing tables
  • ss – Displays socket statistics
  • tcpdump – Captures and analyzes network traffic
  • nmap – Scans networks and identifies open ports and services

Security & Logs Analysis

  • ps – Lists running processes
  • top / htop – Monitors real-time system resource usage
  • journalctl – Views system logs
  • grep – Searches for patterns within text
  • auditctl – Configures auditing rules for system activity monitoring

Other Useful Commands

  • df – Displays disk space usage in a human-readable format
  • mkdir – Create directories
  • touch – Create empty files
  • uname – Displays system information (kernel version, architecture, etc.)
  • kill – Forcefully terminates a process by its Process ID
  • ln – create symlinks (shortcuts) for files or directories
  • ping – Tests network connectivity to a specific host
  • tar – Creates a compressed archive of a directory
  • scp – Securely copies files between machines
  • env – Lists environment variables used by the system

Understanding Linux Permissions

In Linux, every file and folder has some rules about who can do what with it. It's like having toys in a toy box:

  • Owner: The person who owns the toy (file).
  • Group: A group of friends who are allowed to play with it.
  • Others: Everyone else who might want to play.

Each person can have three kinds of permissions:

  • r (read) – You can look at it, like reading a book.
  • w (write) – You can change it, like drawing in a coloring book.
  • x (execute) – You can run it, like pressing a button to start a toy robot.

How Permissions Look

When you type ls -l, you might see something like this: 

-rw-r--r-- 1 alice users 1234 Apr 14 10:00 notes.txt
  • -rw-r--r-- means:
    • rw- – Owner can read & write
    • r-- – Group can only read
    • r-- – Others can only read

Changing Permissions

  • chmod – Changes who can do what with the file
  • chown – Changes who owns the file
  • chgrp – Changes the group of the file

Examples

  • chmod 755 script.sh – Owner can do everything, others can read & run it
  • chmod 600 secret.txt – Only the owner can read & write it
  • chown bob:bob file.txt – Makes Bob the new owner

Understanding the Linux Shell

The shell is a command-line interface (CLI) that allows users to interact with the Linux operating system. It's like a translator between you and the computer, helping execute commands, run scripts, and automate tasks.

What Does a Shell Do?

  • Interprets user commands and executes them
  • Runs scripts to automate tasks
  • Manages processes and system administration tasks
  • Allows navigation through the file system

Common Shells in Linux

Shell Name Description Command to Use
Bash (Bourne Again Shell) Default and most widely used shell on Linux bash
Zsh (Z Shell) Extended version of Bash with better customization zsh
Fish (Friendly Interactive Shell) User-friendly shell with auto-suggestions fish
Dash (Debian Almquist Shell) Lightweight and fast shell used in system scripts dash
Ksh (KornShell) Advanced scripting capabilities ksh

How to Check Your Default Shell?

Run this command in the terminal to see which shell is currently in use:

echo $SHELL

Switching to a Different Shell

To change your shell temporarily, type the shell name:

zsh (Switches to Zsh)

To set a new default shell permanently:

chsh -s /bin/zsh (Changes default shell to Zsh)

  • Check your current shell and experiment with switching shells.
  • Try customizing your shell with themes (like oh-my-zsh for Zsh).
  • Create a basic shell script and execute it using Bash.

Working with Files and Directories

Managing files and directories in Linux is essential for navigating and organizing your system efficiently. Below are key commands used for viewing, editing, and manipulating files.

Viewing and Editing Files

  • cat – Displays the contents of a file in the terminal
  • less / more – Allows scrolling through large files without loading everything at once
  • nano – A simple and user-friendly text editor for editing files
  • vim – A powerful editor with advanced features (optional at this level)

Example: cat file.txt will print the content of file.txt directly in the terminal.

Redirection and Piping

Linux allows powerful output control using redirection and piping:

  • > – Redirects output to a file (overwrites if it exists)
  • >> – Appends output to an existing file instead of overwriting
  • | – Sends output from one command to another command for processing
  • < – Takes input from a file for a command

Example: echo "Hello World" > myfile.txt will create myfile.txt with the text "Hello World".
Example: ls -lah | grep ".txt" will list only files ending with .txt.

Practice Challenge

  • Use cat to view a file and then try less on a large file.
  • Try editing a text file using nano.
  • Redirect command output to a new file using > and append using >>.
  • Pipe ls output into grep to filter for specific file types.

Linux Networking Essentials

Networking is a crucial part of system administration and cybersecurity. Below are key commands used for checking IP addresses, testing connectivity, analyzing DNS queries, and managing system services.

Checking IP Addresses

  • ip a – Displays all network interfaces and assigned IP addresses
  • ifconfig – Shows network interface details (older but still used in many distributions)

Example: ip a will list all IP addresses and interfaces.

Testing Connectivity

  • ping – Tests the reachability of a remote server by sending ICMP requests
  • traceroute – Tracks the path packets take to reach a destination
  • netstat – Displays active network connections and listening ports
  • ss – Alternative to netstat that provides more detailed socket statistics

Example: ping google.com tests whether Google's servers are reachable.

DNS Analysis

  • nslookup – Queries DNS servers to find IP addresses
  • dig – Advanced DNS lookup tool for detailed analysis

Example: nslookup google.com retrieves Google's IP address.

Managing Services

  • systemctl – Manages services on modern Linux distributions (systemd-based)
  • service – Older alternative for controlling system services

Example: systemctl status apache2 checks the status of the Apache web server.

Practice Challenge

  • Find your IP address using ip a or ifconfig.
  • Test connectivity using ping and traceroute.
  • Query DNS records using nslookup and dig.
  • Check the status of a system service using systemctl or service.

Shell Scripting Basics

Shell scripting allows you to automate tasks, create system commands, and streamline workflows. Below are the essentials for getting started.

Writing a Basic Shell Script

A shell script is simply a text file containing commands executed in order. Create a basic script:

#!/bin/bash
echo "Hello, this is my first shell script!"

Save this as script.sh.

Making Scripts Executable

Before running a script, you must give it execution permissions:

chmod +x script.sh

Now, you can run it using:

./script.sh

Using Variables, Conditionals, and Loops (Intro)

  • VARIABLE="Hello" – Defines a variable
  • echo $VARIABLE – Prints the variable
  • if [ condition ]; then ... fi – Basic conditional
  • for var in list; do ... done – Looping structure

Example: Print numbers from 1 to 5 using a loop:

for i in {1..5}; do
  echo "Number: $i"
done

Automating Simple Tasks

Shell scripts are great for automation. Example: Automatically backup a directory:

#!/bin/bash
tar -czf backup.tar.gz /path/to/directory

Run it, and it creates a backup archive!

Practice Challenge

  • Create a simple script that prints your name.
  • Make the script executable and run it.
  • Write a script that checks if a file exists using if condition.
  • Automate a task using a loop (e.g., print numbers).

Basic Security Practices

Securing your Linux system is essential to protect files, prevent unauthorized access, and maintain system integrity. Below are key security practices to follow.

File Permission Best Practices

Linux provides a permission system to control who can read, modify, or execute files. Use the following commands to manage file permissions:

  • ls -l – Displays detailed file permissions
  • chmod – Changes file access permissions
  • chown – Modifies file ownership
  • umask – Defines default permissions for new files

Example: chmod 700 myfile.txt restricts access to only the file owner.

Using sudo Responsibly

The sudo command allows users to execute privileged commands without logging in as root. Proper usage ensures security:

  • Use sudo only when necessary to avoid accidental system changes.
  • Restrict sudo access using the /etc/sudoers file.
  • Monitor sudo activity using logs: sudo cat /var/log/auth.log

Example: sudo apt update updates the system package list.

Difference Between sudo, su, and doas

Command Description Usage Example
sudo Runs a single command with administrative privileges sudo apt install package
su Switches to the root user (or another user) su - (Switch to root shell)
doas Lightweight alternative to sudo (used in OpenBSD and some Linux distributions) doas ls /root

  • Modify a file's permissions using chmod.
  • List your elevated privileges using sudo.
