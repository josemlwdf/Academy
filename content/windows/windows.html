Basic Windows Course

Introduction to Windows

Windows is a widely-used operating system developed by Microsoft. Known for its user-friendly graphical interface, it's the standard OS for personal and enterprise computing across the world.

What is an Operating System?

An operating system (OS) is software that manages computer hardware and software resources while providing services for applications. It allows users to interact with the system via a graphical or command-line interface.

Latest Windows Relevant Versions and Their Differences

  • Windows 7 – Introduced a more refined user interface and improved performance, but is no longer supported.
  • Windows 10 – A major update with enhanced security, virtual desktops, and better integration with cloud services.
  • Windows 11 – Features a redesigned interface, improved gaming and productivity tools, and optimized performance.

In this course, you'll learn how to navigate the Windows environment, use command-line tools (CMD & PowerShell), and understand the basics of the Windows file system and system management.

Desktop vs Server Editions & GUI vs Command Line

Windows offers different editions tailored for personal and enterprise use. Additionally, users can interact with Windows through a graphical interface or command-line tools.

Desktop vs Server Editions

  • Windows Desktop – Designed for personal and business use with a user-friendly interface and support for applications like Microsoft Office.
  • Windows Server – Optimized for managing networks, databases, and enterprise-level applications, offering better security, remote access, and server-specific tools.

Example: A standard office laptop runs Windows 10, whereas a corporate web server operates on Windows Server 2019.

GUI vs Command Line (PowerShell & CMD)

  • GUI (Graphical User Interface) – Uses windows, icons, and menus for user interaction, making it more intuitive.
  • CMD (Command Prompt) – Basic Windows command-line tool used for administrative tasks like file management and networking.
  • PowerShell – Advanced command-line tool offering scripting capabilities, automation, and system management beyond CMD.

Example: Using GUI, you can click on folders to navigate, while in CMD, you use cd to move between directories. PowerShell allows running scripts to automate tasks efficiently.

Practice Challenge

  • Check your Windows edition using winver in CMD.
  • Open PowerShell and run Get-ComputerInfo to explore system details.
  • Compare using File Explorer (GUI) vs CMD to navigate folders.

Windows File System & Security Essentials

Understanding the Windows file system is crucial for cybersecurity professionals. Windows organizes data in drives, with C:\ being the primary system drive where critical files and configurations are stored.

Lesson 1: Root Drive C:\

The root drive contains core system files and essential directories. Cybersecurity professionals monitor this area for unauthorized modifications.

  • C:\Windows\System32 – Contains critical system executables; often targeted by malware.
  • C:\Users\Public – Shared space between users, prone to unauthorized data leakage.
  • C:\Temp – Used by applications for temporary storage; often exploited for malicious file execution.

Lesson 2: C:\Users (User Profiles & Security)

Each user has a personal directory under C:\Users, where sensitive personal files and configurations are stored.

  • C:\Users\Username\AppData\Roaming – Contains user-specific application data, often leveraged in persistence mechanisms.
  • C:\Users\Username\Downloads – A frequent location for malware execution; ensure files are verified before opening.
  • C:\Users\Username\Desktop – Attackers may store malicious shortcuts or scripts in this directory.

Lesson 3: C:\Windows (Critical System Directory)

The Windows directory hosts OS-specific configurations and executable files. Unauthorized modifications here can indicate system compromise.

  • C:\Windows\System32\drivers\etc\hosts – Often modified by malware to redirect network requests.
  • C:\Windows\Prefetch – Useful for forensic analysis of recent program executions.
  • C:\Windows\SysWOW64 – System directory for 32-bit applications on 64-bit Windows.

Lesson 4: Program Files / Program Files (x86) (Software Security)

This directory contains installed applications. Attackers may abuse this location for persistence or privilege escalation.

  • C:\Program Files – Default storage for 64-bit applications.
  • C:\Program Files (x86) – Stores 32-bit applications.
  • Monitor software installed here for unauthorized or unexpected programs.

Cybersecurity Practice Challenge

  • Identify system-critical files in C:\Windows\System32 using dir /s or ls (WSL).
  • Analyze the C:\Windows\Prefetch directory for traces of recent executions.
  • Check user profiles in C:\Users for suspicious files using dir /A to reveal hidden folders.
  • Monitor installed applications in Program Files for unauthorized software.

Essential Windows Security Commands

These Windows command-line tools help analyze security threats, manage access controls, and monitor system integrity.

System Security & Information Gathering

  • systeminfo – Displays detailed system information, including OS version, installed updates, and hardware details.
  • wmic – A powerful Windows Management Instrumentation tool for retrieving system details, managing processes, and monitoring services.
  • hostname – Returns the computer’s network name, useful for identifying systems in a network.
  • whoami – Shows the current user logged into the system along with privilege details.
  • net user – Lists all local user accounts and their status, helping administrators monitor account security.

Network Security & Threat Analysis

  • ipconfig – Displays network adapter configurations, including assigned IP addresses, DNS settings, and active connections.
  • netstat – Shows active network connections, open ports, and associated processes, helping detect unauthorized activity.
  • arp – Displays the ARP table, which maps IP addresses to MAC addresses, useful for detecting MITM (Man-in-the-Middle) attacks.
  • route – Displays the system’s routing table, showing how network traffic is directed.
  • tracert – Traces the route packets take to reach a destination, identifying connection issues or network interception points.

Process & Malware Detection

  • tasklist – Lists all running processes along with their memory usage and associated services.
  • taskkill – Allows the termination of running processes, useful for stopping malicious or unresponsive applications.
  • Get-EventLog – Retrieves event logs related to security, helping analyze system activity for unauthorized changes.
  • Get-WmiObject – Queries system processes and services using Windows Management Instrumentation (WMI), often used for forensic investigations.

File System Integrity & Incident Response

  • dir – Lists the contents of directories, including files and subdirectories.
  • attrib – Modifies file attributes, such as hiding files or restoring access to restricted files.
  • Get-FileHash – Computes a cryptographic hash of a file, useful for verifying file integrity and detecting tampering.
  • icacls – Displays and modifies file permissions, helping secure sensitive data.
  • cipher – Securely wipes free disk space to prevent the recovery of deleted files.

Windows Log & Forensic Investigation

  • wevtutil – Retrieves and manages Windows Event logs, essential for analyzing security incidents.
  • auditpol – Displays and configures audit policies, helping track system integrity and unauthorized access attempts.
  • fsutil – Provides advanced file system utilities, such as retrieving NTFS change journal entries.
  • schtasks – Lists scheduled tasks, helping detect persistent threats using automated executions.
  • Get-WinEvent – Queries Windows security logs to identify signs of compromise or suspicious activity.

Managing Files and Folders in Windows

Understanding how to create, rename, move, and delete files or folders is fundamental to file management. Windows provides multiple ways to handle files efficiently using both graphical and command-line tools.

Creating, Renaming, Moving, and Deleting Files/Folders

  • mkdir foldername – Creates a new folder
  • ren oldname newname – Renames a file or folder
  • move source destination – Moves a file or folder
  • del filename – Deletes a file
  • rmdir foldername – Deletes an empty folder

Example: mkdir Documents creates a folder named "Documents."

File Extensions and Types

File extensions indicate the format and usage of a file.

  • .txt – Plain text file
  • .jpg, .png – Image files
  • .exe – Executable program
  • .docx, .xlsx – Microsoft Office documents
  • .zip – Compressed archive file

Example: A file named report.docx is a Word document.

Copy, Cut, and Paste Operations

  • copy source destination – Copies a file or folder to another location
  • xcopy /s source destination – Copies directories and their contents
  • Ctrl + C – Copy (GUI)
  • Ctrl + X – Cut (GUI)
  • Ctrl + V – Paste (GUI)

Example: copy notes.txt D:\Backup copies "notes.txt" to a Backup folder on drive D.

Hidden/System Files and Folder Options

  • attrib +h filename – Hides a file
  • attrib -h filename – Unhides a file
  • dir /a:h – Lists hidden files
  • Enable "Show hidden files" in File Explorer to view hidden files

Example: attrib +h secret.txt hides "secret.txt" from normal view.

Practice Challenge

  • Create a folder using mkdir and place a file inside.
  • Rename a file using ren.
  • Copy and move a file using copy and move.
  • Try hiding a file using attrib +h, then unhide it.

Creating and Switching Users

Windows allows multiple user accounts to manage access and permissions efficiently. Below are key commands and methods for creating and switching users.

Creating a New User Account

  • net user username password /add – Creates a new user account.
  • Control Panel > User Accounts – Allows graphical user management.
  • Settings > Accounts > Family & Other Users – Adds new accounts in modern Windows versions.

Example: net user JohnDoe mypassword /add adds a user named "JohnDoe".

Switching Users

  • shutdown /l – Logs out the current user.
  • Ctrl + Alt + Delete > Switch User – Graphically switches users.
  • Fast User Switching – Allows switching between accounts without logging out completely.

Administrator vs Standard Accounts

  • Administrator Account – Has full system access, can install programs, change settings, and manage security configurations.
  • Standard Account – Restricted access, used for daily activities while preventing unauthorized system changes.
  • Guest Account – Temporary account with minimal permissions (disabled by default).

Example: Administrators can install software, while standard users cannot modify critical system settings.

Practice Challenge

  • Create a new user using net user.
  • Switch between users using graphical and command-line methods.
  • Check your account type using net user username.

Viewing IP Address & File Sharing on LAN

Managing network settings in Windows involves checking IP addresses and setting up file sharing for local networks (LAN). Below are the essential methods.

Viewing IP Address

  • ipconfig – Displays network configuration details, including assigned IP addresses.
  • ipconfig /all – Provides detailed information about all network adapters.
  • GUI Method: Open Settings > Network & Internet > Status and click on "Properties" for your active connection.

Example: Running ipconfig in Command Prompt shows your IPv4 address.

File Sharing Basics on LAN

  • Enable Network Discovery in Control Panel > Network and Sharing Center.
  • Right-click a folder > Select Properties > Go to the Sharing tab > Click "Share" to allow access.
  • Use \\ComputerName\SharedFolder in File Explorer to access shared folders on another LAN-connected PC.
  • For advanced sharing, configure permissions in Advanced Sharing settings.

Example: Sharing a folder allows multiple users on the same network to access files easily.

Sharing a Folder from CMD

  • net share ShareName=C:\Path\To\Folder /grant:UserName,full – Shares a folder with specific user access.
  • net share – Lists all currently shared folders.
  • net share ShareName /delete – Stops sharing a specific folder.

Example: net share PublicFiles=C:\Users\Public /grant:Everyone,read shares the Public folder with read-only access.

Accessing a Shared Folder from CMD

  • net use Z: \\ComputerName\SharedFolder /persistent:yes – Maps the shared folder as a network drive.
  • dir \\ComputerName\SharedFolder – Lists the files inside a shared folder.
  • net use Z: /delete – Disconnects the mapped network drive.

Example: net use Z: \\OfficePC\Documents /persistent:yes maps the shared "Documents" folder from a computer named "OfficePC."

Discovering Shared Folders on a Host via CMD

  • net view \\ComputerName – Lists all shared resources on a specific computer.
  • net view – Displays available shared computers on the network.
  • wmic share get name,path – Retrieves a list of shared folders and their locations.

Example: Running net view \\Workstation1 shows all shared folders on "Workstation1."

Practice Challenge

  • Check your IP address using both ipconfig and the GUI method.
  • Share a folder via CMD using net share and confirm it using net view.
  • Access a shared folder using net use and navigate its contents.
  • Discover shared resources on a LAN-connected device using net view \\ComputerName.

CMD vs PowerShell & Automation

The Windows Command Prompt (CMD) and PowerShell are powerful tools used for system administration and automation.

Opening CMD and PowerShell

  • Press Win + R, type cmd, and press Enter to open Command Prompt.
  • Press Win + R, type powershell, and press Enter to open PowerShell.
  • Right-click the Start menu and select Command Prompt or PowerShell.
  • Search for cmd or PowerShell in the Windows Start menu.

Difference Between CMD and PowerShell

Feature CMD PowerShell
Purpose Basic command-line interface for executing system commands Advanced scripting environment for automation and system management
Syntax Uses simple DOS-style commands (e.g., dir, copy) Uses objects and scripting (e.g., Get-Process, Set-ExecutionPolicy)
Script Support Runs basic batch files (.bat) Supports complex automation scripts (.ps1)
Usage Common for quick tasks, basic file management Used by administrators for automation, configuration, and system management

Simple Automation with Batch and PowerShell Scripts

Batch Script Example: Auto-Creating a Folder

Save this as create_folder.bat:

@echo off
mkdir C:\MyFolder
echo Folder Created Successfully!
pause

Double-click the file to run it.

PowerShell Script Example: Checking System Uptime

Save this as check_uptime.ps1:

$uptime = (Get-CimInstance Win32_OperatingSystem).LastBootUpTime
Write-Host "System last restarted on: $uptime"

Run this in PowerShell using powershell -ExecutionPolicy Bypass -File check_uptime.ps1.
